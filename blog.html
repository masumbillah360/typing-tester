Event Loop in JavaScript

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks
In a thread, the function at the top of the stack is what's currently running. The event loop is at the bottom of the stack, or at least it's underneath all JavaScript functions, so it only gets to run when all previously running JavaScript functions have returned or thrown. Event Loop first workes for function or event who stay on stack. When stacked function or event finish then workes for function or event who stored in Queue. How many way to get "undefined" in JavaScript

1. Uninitialized variable : A declared variable but not yet assigned with a value (uninitialized) is by default undefined.

2. Function with no return : A function that doesn't have return statement implicitly returns undefined

3. Function parameters : Perameter that is not passed will be undefined.

4. If return has nothing on the right side will be return undefined.

5. Accessing a non-existing property : When accessing a non-existing object property, JavaScript returns undefined.

6. undefined in arrays : You get undefined when accessing an array element with an out of bounds index.

7. Deleting an element inside of array they return and undefined for deleted element

8. If you set a value directly "undefined" you got undefined
